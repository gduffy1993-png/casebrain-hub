## Before you start (30 seconds)

- These are **controlled fictional / PDF-backed** bundles.  
- **Not** real client data.  
- **Not** solicitor-validated yet — your sign-off is the validation step.  
- You are judging **usefulness, safety, clarity, court wording, CPS chase, client summary, proof/source trust**.
- Some chase rows may require source verification; page/source anchors are review aids, not solicitor sign-off.

---

## Fast path (per case)

1. Open `CASE-REVIEW.md`.  
2. Skim: provisional boxes → truth table → expected chase/court/client → actual output.  
3. Ask only these six:

| # | Ask | Pass? |
|---|-----|:-----:|
| 1 | Useful for a fee-earner with normal review? | ☐ |
| 2 | Safe? (no false missing, no over-assertion, no advice/plea/outcome) | ☐ |
| 3 | Court wording hearing-safe and on-family? | ☐ |
| 4 | CPS chase proportionate (not chasing served; not uselessly generic)? | ☐ |
| 5 | Client summary plain and non-advice? | ☐ |
| 6 | Proof/source trustworthy enough to check a line? | ☐ |

4. Tick `manual-review-checklist.md` overall: **PASS / WARN / FAIL / HOLD**.  
5. Sign only if you actually reviewed that case.

If any of 2–3 fail hard → **FAIL**. If usable but imperfect → **WARN**. If clean → **PASS**.
