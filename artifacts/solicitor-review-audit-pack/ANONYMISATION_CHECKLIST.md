# Anonymisation Checklist

> **Before any bundle is shared for solicitor-reviewed audit.**  
> **Do not upload real client data unless your firm has explicitly authorised this audit and secure handling is in place.**

---

## Identity and contact

| Item | Redact / replace | Done |
|------|------------------|------|
| Defendant full name | Use e.g. `ANON_DEF_001` | ☐ |
| Co-defendant names | Anonymised labels | ☐ |
| Complainant / witness names | Anonymised or role labels | ☐ |
| Home / work addresses | Remove or fictionalise | ☐ |
| Dates of birth | Remove or shift (consistent offset) | ☐ |
| Phone numbers | Remove or fictionalise | ☐ |
| Email addresses | Remove or fictionalise | ☐ |
| Vehicle registrations | Remove or fictionalise | ☐ |
| Social media handles (live) | Remove or fictionalise | ☐ |

---

## Case and court identifiers

| Item | Redact / replace | Done |
|------|------------------|------|
| URN / case reference | Remove or fictionalise | ☐ |
| Firm file number | Remove | ☐ |
| Court name (if identifying) | Generic e.g. “Crown Court A” | ☐ |
| Solicitor / counsel names on file | Remove unless reviewer is same firm | ☐ |
| Hearing dates (if identifying) | Shift or remove | ☐ |

---

## Sensitive material

| Item | Action | Done |
|------|--------|------|
| Victim / child identity | Strict redaction | ☐ |
| Sexual offence detail beyond review need | Minimise; role labels only where possible | ☐ |
| Medical / psychiatric records | Redact identifiers; consider separate secure channel | ☐ |
| Images of persons | Redact faces/plates unless essential and authorised | ☐ |
| Domestic address in screenshots/maps | Redact | ☐ |

---

## Digital and metadata

| Item | Action | Done |
|------|--------|------|
| PDF metadata (author, title, paths) | Strip or sanitise | ☐ |
| File names (`Smith_Bundle.pdf`) | Rename neutral | ☐ |
| Email headers (if in bundle) | Redact addresses | ☐ |
| Phone extraction metadata (IMEI, IMSI) | Redact or fictionalise | ☐ |
| GPS / cellsite maps with home location | Redact | ☐ |
| Watermarks (firm / CPS / vendor) | Remove if identifying | ☐ |

---

## Legal and privilege

| Item | Action | Done |
|------|--------|------|
| Privileged material | **Do not include** unless expressly in scope | ☐ |
| LPP correspondence | Exclude | ☐ |
| Without-prejudice material | Exclude | ☐ |
| Third-party confidentiality markings | Respect; do not republish beyond audit | ☐ |

---

## Reviewer handling

| Item | Action | Done |
|------|--------|------|
| Bundle stored on approved channel only | ☐ |
| Truth key stored without re-pasting live identifiers | ☐ |
| No forwarding to personal email / consumer AI tools | ☐ |
| Delete local copies when review complete (if required) | ☐ |

---

## Fictional training bundles

If using **fully fictional** bundles (no real client source):

- Confirm no accidental paste of real names from other files  
- Label bundle `fictional_training` in truth key  
- Still avoid real third-party identifiers in examples
