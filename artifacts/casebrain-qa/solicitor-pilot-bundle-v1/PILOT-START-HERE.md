# Pilot start here

**Material:** controlled fictional gold (Waves A+B)  
**Not:** live client files · legal advice · autonomous sending

---

## 10-minute path

1. Read the claim line in `README.md` (30 seconds).  
2. Open `WAVE-A-B-CASE-INDEX.md`.  
3. Pick **one** case — suggested: **CASE-01** (phone / attribution) or **CASE-08** (charge mismatch).  
4. Open `cases/CASE-XX/CASE-REVIEW.md`. Skim: risk focus → court line → chase → do-not → client (if present) → hard safety.  
5. Tick gut feel on `REVIEWER-FEEDBACK-FORM.md`: useful before court? unsafe? overall PASS/WARN/FAIL.

Stop. That is enough for a first impression.

---

## 30-minute path

1. Do the 10-minute path on **CASE-01**.  
2. Add **CASE-02** (BWV referred-only) and **CASE-04** (CCTV stills vs master).  
3. Skim **CASE-15** (s172 / SJP) or **CASE-20** (OCR / listing date) if motoring or listing risk matters to you.  
4. Read `TRUST-AND-SAFETY-SUMMARY.md` (2–3 minutes).  
5. Complete one feedback form per case reviewed (or one form covering the sample with case IDs listed).  
6. Optional: `PILOT-OFFER-SUMMARY.md` if you are deciding on a 30-day firm pilot.

---

## What feedback to give

Use `REVIEWER-FEEDBACK-FORM.md`. We care most about:

| Question | Why |
|----------|-----|
| Useful before court? | Practical value |
| Unsafe / misleading? | Hard stop if yes |
| Court wording realistic? | Hearing-safe drafts |
| CPS chase usable? | Accept / light edit / rewrite |
| Client summary clear? | Provisional, not advice |
| Proof / source receipts useful? | Honesty vs fake certainty |
| Time saved? | First-pass triage |
| Blockers? | Access, noise, trust |

Mark **FAIL** and stop promotion if you see plea/outcome advice, invented “served” evidence, or wrong-defendant bleed.
